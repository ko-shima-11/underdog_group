# 施設利用申請システム モックUI

## 📖 プロジェクト概要
施設内の部屋を申請・承認・管理するためのWebアプリケーションのモックアップです。  
3つのユーザーロール（申請者・承認者・施設管理者）それぞれに最適化された画面を提供します。

## 🎯 主な特徴

### 1. ロール別の初期画面最適化
各ユーザーの立場や利用頻度に応じて、最初に表示される画面を切り替えます。

- **申請者**: 申請フォーム画面
- **承認者**: 申請の承認・拒否画面
- **施設管理者**: 通知一覧と週間カレンダー画面

### 2. スワイプ操作による効率的な承認フロー
承認者（先生）の時間を最小化するため、マッチングアプリのようなスワイプUIを採用。

- **右スワイプ → 承認**
- **左スワイプ → 拒否**
- 一発で判定できる直感的な操作

> 参考: [Slackの新機能にみる「スワイプUI」の効果](https://note.com/ritar/n/n9e960988c8c8)

### 3. 週間カレンダー表示
施設管理者が各施設の1週間の使用状況を一目で把握できるカレンダービュー。

- 施設ごとの切り替え表示
- 時間帯別の空き状況の可視化（埋まり・空き・未使用）

## 🎨 UI/UXの工夫

- **白ベースの配色**: 清潔感のある明るいデザイン
- **必須項目の明示**: アスタリスク（*）で必須入力を視覚化
- **リアルタイムプレビュー**: 入力内容を即座に確認可能
- **レスポンシブ対応**: モバイル・タブレット・デスクトップに対応
- **バリデーション**: 入力エラーの視覚的フィードバック

## 📁 ファイル構成

```
├── index.html          # メインHTML
├── style.css           # スタイルシート
├── main.js             # メインJavaScript
├── modules/
│   ├── applicant.js   # 申請者画面
│   ├── approver.js    # 承認者画面
│   └── manager.js     # 施設管理者画面
├── Requets.md          # 要件定義
└── README.md           # このファイル
```

## 🚀 使い方

1. `index.html` をブラウザで開く
2. ヘッダーのロール切り替えボタンで各画面を確認
   - 申請者
   - 承認者
   - 施設管理者
